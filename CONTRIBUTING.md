# スキル作成・投稿ガイド

このガイドでは、Claude Code用スキルの作成方法と投稿手順を説明します。

## 📝 スキルファイルの基本構造

```markdown
---
description: スキルの短い説明（1行）
---

# スキル名

## 概要
このスキルの目的と機能を説明

## 前提条件
- 必要なツールやセットアップ（あれば）

## 使い方
引数がある場合は $ARGUMENTS で受け取れます。
例: /my-skill ファイル名.txt

## 実行手順

1. 最初のステップ
2. 次のステップ
...

## 注意事項
- 注意点があれば記載
```

## 🎯 よいスキルを書くためのポイント

### 1. 明確な目的を持つ

```markdown
# Good
このスキルは、Gitのコミットメッセージを Conventional Commits 形式で生成します。

# Bad
このスキルはGit関連の作業を手伝います。
```

### 2. 具体的な手順を書く

```markdown
# Good
1. git diff --staged を実行して変更内容を確認
2. 変更の種類（feat/fix/docs等）を判断
3. 変更内容を要約してコミットメッセージを生成
4. ユーザーに確認後、git commit を実行

# Bad
1. 変更を確認
2. コミットする
```

### 3. $ARGUMENTS を活用する

ユーザーからの入力を受け取る場合：

```markdown
## 使い方
/translate-code $ARGUMENTS

$ARGUMENTS には変換先の言語を指定します。
例: /translate-code TypeScript

引数がない場合はユーザーに確認します。
```

### 4. 安全性を考慮する

破壊的な操作には確認ステップを含める：

```markdown
## 手順
1. 削除対象のファイルを一覧表示
2. **ユーザーに確認を求める**（削除していいか明示的に聞く）
3. 確認が取れたら削除を実行
```

## 📁 ファイル命名規則

- **ファイル名**: `kebab-case.md`（小文字、ハイフン区切り）
- **コマンド名**: ファイル名がそのままコマンド名になる

```
# Good
review-code.md      → /review-code
generate-test.md    → /generate-test

# Bad
ReviewCode.md       → 避ける
generate_test.md    → アンダースコアは避ける
```

## 📂 カテゴリの選び方

| カテゴリ | 配置先 | 用途 |
|---------|--------|------|
| development | `skills/development/` | コーディング、テスト、デバッグ |
| documentation | `skills/documentation/` | ドキュメント作成、更新 |
| productivity | `skills/productivity/` | Git操作、タスク管理、ワークフロー |
| creative | `skills/creative/` | デザイン、アート、クリエイティブ作業 |
| utilities | `skills/utilities/` | ファイル操作、変換、汎用ツール |

迷ったら最も近いカテゴリを選んでください。

## ✅ 投稿前チェックリスト

- [ ] ファイル名が kebab-case になっている
- [ ] 概要・使い方・手順が明確に書かれている
- [ ] 適切なカテゴリに配置されている
- [ ] 破壊的な操作には確認ステップがある
- [ ] 機密情報（APIキー等）を含んでいない
- [ ] 実際にClaude Codeで動作確認済み

## 🔧 テンプレート

`examples/` ディレクトリにテンプレートがあります：

- `skill-template.md` - 基本的なスキルテンプレート
- `skill-with-args.md` - 引数を受け取るスキルのテンプレート

## 💡 スキルのアイデア

作成するスキルに迷ったら：

- 繰り返し行う作業を自動化
- プロジェクト固有のルールを標準化
- チームの知見をスキル化
- 既存のワークフローを改善

## 🚀 投稿方法

1. このリポジトリをフォーク
2. 適切なカテゴリにスキルファイルを追加
3. Pull Requestを作成
4. レビュー後マージ
