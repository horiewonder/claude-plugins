# Generate Test

## 概要

指定されたファイルまたは関数に対するユニットテストを生成します。

## 使い方

```
/generate-test <ファイルパス または 関数名>
```

例:
```
/generate-test src/utils/parser.ts
/generate-test calculateTotal
```

## 実行手順

1. $ARGUMENTS で指定された対象を確認
   - ファイルパスの場合: そのファイルを読み込む
   - 関数名の場合: プロジェクト内で該当関数を検索
2. 対象のコードを分析:
   - 関数のシグネチャ（引数、戻り値）
   - 正常系・異常系のパターン
   - エッジケース
3. プロジェクトのテストフレームワークを検出（Jest, Vitest, pytest など）
4. テストコードを生成:
   - 正常系テスト
   - 境界値テスト
   - エラーケーステスト
5. 生成したテストをユーザーに提示
6. 承認されたらテストファイルを作成

## テスト配置ルール

- 既存のテストディレクトリ構造に従う
- なければ `__tests__/` または `*.test.ts` 形式を提案

## 注意事項

- 引数がない場合は、テスト対象を確認する
- 既存のテストファイルがある場合は上書き確認を行う
