# Quick Commit

## 概要

ステージングされた変更を確認し、Conventional Commits形式のコミットメッセージを生成してコミットします。

## 使い方

```
/quick-commit
```

## 実行手順

1. `git status` で現在の状態を確認
2. `git diff --staged` でステージングされた変更内容を確認
3. 変更内容を分析して、適切なコミットタイプを判断:
   - `feat`: 新機能
   - `fix`: バグ修正
   - `docs`: ドキュメントのみの変更
   - `style`: コードの意味に影響しない変更（空白、フォーマットなど）
   - `refactor`: バグ修正でも機能追加でもないコード変更
   - `test`: テストの追加・修正
   - `chore`: ビルドプロセスやツールの変更
4. Conventional Commits形式でコミットメッセージを生成
5. 生成したメッセージをユーザーに提示して確認を求める
6. 承認されたら `git commit` を実行

## コミットメッセージ形式

```
<type>(<scope>): <subject>

<body>
```

## 注意事項

- ステージングされた変更がない場合は、その旨を伝えて終了
- コミットメッセージは簡潔に、50文字以内を目標に
